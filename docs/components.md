## SEO

**File:** `src/lib/components/ui/SEO.svelte`

Handles all `<head>` metadata for a page. Uses `<svelte:head>` internally ‚Äî just drop it into any route's template.

### Props

```ts
interface Props {
  title: string;           // Page title ‚Äî auto-suffixed with "‚Äî Kidus Dereje" unless it equals the site name
  description: string;     // Meta description + OG/Twitter description
  type?: 'website' | 'article';  // default: 'website'
  publishedAt?: string;    // ISO date ‚Äî only used when type === 'article'
  updatedAt?: string;      // ISO date ‚Äî only used when type === 'article'
  tags?: string[];         // Used as article:tag meta tags + OG image generation
  coverImage?: string;     // Absolute path or full URL ‚Äî overrides auto-generated OG image
}
```

### What it generates

- `<title>` ‚Äî `"Page Title ‚Äî Kidus Dereje"` (or just `"Kidus Dereje"` for the home page)
- `<meta name="description">`
- `<link rel="canonical">` ‚Äî derived from `$page.url.href` (always correct, no manual input needed)
- Open Graph: `og:title`, `og:description`, `og:url`, `og:type`, `og:image`, `og:site_name`
- Twitter Card: `summary_large_image` with title, description, image
- Article tags (when `type="article"`): `article:published_time`, `article:modified_time`, `article:author`, `article:tag` per tag

### OG image

When `coverImage` is not set, the OG image is auto-generated by routing to `/og?title=...&description=...&type=...&tags=...`. This is a dynamic image generation endpoint.

When `coverImage` is provided, it is used directly. Relative paths (starting with `/`) are absolutified to `https://kidus.dev/...`.

### Usage

```svelte
<!-- Portfolio page -->
<SEO title="Kidus Dereje" description="Founding Engineer, researcher..." />

<!-- Blog post (in BlogPostLayout.svelte) -->
<SEO
  title={title}
  description={description}
  type="article"
  publishedAt={publishedAt}
  updatedAt={updatedAt}
  tags={tags}
  coverImage={coverImage}
/>
```

## VantaBackground

**File:** `src/lib/components/animation/VantaBackground.svelte`

Wraps Vanta.js WebGL ambient animations. Used to add visual life to pages without distracting from content.

### Props

```ts
interface Props {
  effect: 'GLOBE' | 'NET';
  opacity?: number;       // default: 1 for GLOBE, 0.35 for NET
  interactive?: boolean;  // default: true for GLOBE, false for NET ‚Äî enables mouse/touch tracking
  size?: number;          // NET only ‚Äî point size (default ~1.0, smaller = less prominent)
  maxDistance?: number;   // NET only ‚Äî max connection distance in px (default ~150)
}
```

### Effects

**GLOBE** ‚Äî rotating wireframe globe. Used in the home page hero section. Positions itself as `position: absolute; inset: 0` inside a `position: relative` container. Mouse-interactive: the globe rotates in response to cursor movement.

**NET** ‚Äî animated node-and-line network. Used on the About and Blog listing pages as a full-viewport ambient background. Positions itself as `position: fixed; inset: 0; z-index: -1` so it sits behind all page content without affecting layout.

### Theme reactivity

`VantaBackground` subscribes to `themeStore` and reinitializes the WebGL effect with correct colors on every theme change:

| Theme | Effect | `backgroundColor` | `color` | `color2` |
|---|---|---|---|---|
| Dark | GLOBE | `0x0a0a0a` | `0xF05924` (ember) | `0xffffff` |
| Light | GLOBE | `0xffffff` | `0x2B5CE6` (royal blue) | `0xF05924` |
| Dark | NET | `0x0a0a0a` | `0xF05924` | ‚Äî |
| Light | NET | `0xffffff` | `0x2B5CE6` | ‚Äî |

### SSR safety

`three` and the Vanta effect modules are dynamically imported **inside `onMount`** ‚Äî they never run on the server. A `callId` integer is incremented on each subscriber call; each async branch checks `if (myId !== callId) return` before using the import result, preventing race conditions when the theme switches before a previous import resolves.

### Usage

```svelte
<!-- Home hero (inside a position: relative container) -->
<section class="hero" style="position: relative; overflow: hidden;">
  <VantaBackground effect="GLOBE" />
  ...
</section>

<!-- About / Blog listing (full-viewport background) -->
<VantaBackground effect="NET" opacity={0.3} interactive={false} />
```

### `aria-hidden`

The container div has `aria-hidden="true"` ‚Äî purely decorative, not announced to screen readers.

## ContactModal

**File:** `src/lib/components/ui/ContactModal.svelte`

A self-contained contact form that renders a `.btn--primary` trigger button and a GSAP-animated modal. Submits to `POST /api/contact` via `fetch`. Keeps the destination email address and API key entirely server-side.

### Props

```ts
interface Props {
  label?: string;  // Button label text ‚Äî default: "Get in touch"
}
```

### States

```ts
type Status = 'idle' | 'loading' | 'success' | 'error';
```

- **idle** ‚Äî form ready to fill out
- **loading** ‚Äî fetch in flight; submit button disabled and shows "Sending‚Ä¶"
- **success** ‚Äî confirmation message shown; form inputs hidden; "Close" button resets to idle on close
- **error** ‚Äî inline error message shown below the form; form remains editable

### Form fields

| Field | Type | Validation |
|---|---|---|
| Name | `<input type="text">` | required |
| Email | `<input type="email">` | required, format validated server-side |
| Subject | `<input type="text">` | required |
| Message | `<textarea rows="5">` | required |

Name and Email are displayed side-by-side on ‚â•540px viewports (CSS grid `1fr 1fr`), stacked on mobile.

### GSAP animation

Same pattern as `PublicationModal`:
- **Open:** `setTimeout(0)` flush ‚Üí overlay `opacity 0‚Üí1` (0.35s `power2.out`) + card `scale 0.85‚Üí1, y 20‚Üí0` (`back.out(1.4)`, 0.45s)
- **Close:** card `scale 1‚Üí0.9, opacity 1‚Üí0, y 0‚Üí10` (0.25s `power2.in`) + overlay fade; `open = false` only in `onComplete` to preserve exit animation; status reset to `'idle'` in same callback

Single `activeTl` variable ‚Äî always killed before creating a new timeline.

### Portal

Uses the same inline `portal` action as `PublicationModal` ‚Äî appends the overlay to `document.body` so `position: fixed` isn't constrained by any ancestor `transform` context (e.g. from `use:revealOnScroll` on a parent).

### API contract

```ts
// Request
POST /api/contact
Content-Type: application/json
{ name: string, email: string, subject: string, message: string }

// Success
200 { success: true }

// Validation error
400 { error: string }

// Rate limited (3 req / 15 min per IP)
429 { error: "Too many requests. Please wait before trying again." }

// Send failure
500 { error: "Failed to send. Please try again." }
```

### Usage

```svelte
<!-- Replace a mailto: link with this -->
<ContactModal />

<!-- Custom button label -->
<ContactModal label="Say hello" />
```

The component renders its own trigger button with `use:cursorTarget={'hover'}` and `use:magnetic` ‚Äî do not wrap it in another button.

## PublicationModal

**File:** `src/lib/components/ui/PublicationModal.svelte`

### Props

```ts
interface Props {
  pub: Publication;   // { title, venue, year, url, officialUrl?, bullets[] }
  index: number;      // position in the parent list ‚Äî used for store coordination
}
```

### Behaviour

- Renders a trigger button (`<button class="pub-trigger">`) that opens the modal on click
- Modal state is owned locally with `$state(open)`
- Opening: `open = true` ‚Üí `setTimeout(0)` to flush DOM ‚Üí GSAP timeline animates overlay + card + bullets
- Closing: GSAP exit timeline ‚Üí `open = false` inside `onComplete` (preserves exit animation)
- Escape key handled via `window.addEventListener('keydown', ...)` registered in `onMount`, torn down on destroy
- One open modal at a time: `activePublicationIndex` writable store (`src/lib/stores/publications.ts`) is set to `index` on open; all other instances subscribe and call `closeModal()` if their index no longer matches

### Portal pattern

The `{#if open}` overlay div uses a local `portal` Svelte action:

```ts
function portal(node: HTMLElement) {
  document.body.appendChild(node);
  return {
    destroy() {
      if (node.isConnected) node.remove();
    }
  };
}
```

**Why:** Parent publication list items have `use:revealOnScroll` which applies a GSAP `translateY` transform. Any in-progress or completed (but not `clearProps`-ed) transform on an ancestor creates a CSS stacking context, causing `position: fixed` children to be positioned relative to that ancestor instead of the viewport. Moving the overlay to `document.body` escapes all ancestor transforms.

**Order matters:** `use:portal` is placed before `bind:this={overlayEl}` in the markup ‚Äî Svelte processes `bind:this` first, so `overlayEl` is set before the portal action runs, keeping the GSAP reference valid after the node is reparented.

### GSAP timeline pattern

```ts
let activeTl: gsap.core.Timeline | null = null;

function openModal() {
  bulletEls = [];      // reset before {#if} renders new <li> elements
  open = true;
  activePublicationIndex.set(index);
  setTimeout(() => {
    activeTl?.kill();
    activeTl = gsap.timeline();
    activeTl
      .fromTo(overlayEl, { opacity: 0 }, { opacity: 1, ... })
      .fromTo(cardEl, { opacity: 0, scale: 0.85, y: 20 }, { opacity: 1, scale: 1, y: 0, ease: 'back.out(1.4)' }, '<0.05')
      .fromTo(bulletEls, { opacity: 0, y: 12 }, { opacity: 1, y: 0, stagger: 0.07 }, '-=0.15');
  }, 0);
}

function closeModal() {
  activeTl?.kill();
  activeTl = gsap.timeline({
    onComplete: () => { open = false; }
  });
  activeTl
    .to(cardEl, { opacity: 0, scale: 0.9, y: 10, ease: 'power2.in' })
    .to(overlayEl, { opacity: 0, ease: 'power2.in' }, '<');
}
```

Single `activeTl` variable ‚Äî always killed before creating a new one to prevent competing animations.

### Cursor integration

`PublicationModal` uses `use:cursorTarget={'hover'}` on the trigger button, close button, arXiv link, and Read Publication link. All of these elements also have explicit `cursor: none` in their CSS ‚Äî without this the user-agent `cursor: pointer` would show through even though the site-wide reset is in the universal selector.

The modal overlay (`position: fixed`, `z-index: 9999`) is portaled to `document.body`. `CustomCursor` is at `z-index: 10000` so it renders above the overlay.

### Link buttons

The modal renders one or two link buttons below the title, wrapped in a `div.modal-links` flex row:

- **arXiv button** (`.modal-link--ghost`): always rendered from `pub.url` ‚Äî outline accent style
- **Read Publication button** (`.modal-link--primary`): only rendered when `pub.officialUrl` is set ‚Äî filled accent style, links to the officially published version (e.g. ACM DL, IEEE Xplore, Springer)

```svelte
<div class="modal-links">
  <a href={pub.url} class="modal-link modal-link--ghost">arXiv ‚Üí</a>
  {#if pub.officialUrl}
    <a href={pub.officialUrl} class="modal-link modal-link--primary">Read Publication ‚Üí</a>
  {/if}
</div>
```

To add a "Read Publication" button to a paper, set `officialUrl` in its `Publication` entry. See [`docs/managing-content.md`](./managing-content.md) for the data format.

### Svelte 5 notes

- `bind:this` targets inside `{#if}` blocks must be `$state()` to avoid `non_reactive_update` warnings
- `bulletEls: HTMLLIElement[] = $state([])` ‚Äî reset to `[]` before `open = true` so the array is fresh when the new `<li>` elements bind
- Overlay needs `tabindex="-1"` (role="dialog") and `onkeydown` handler alongside `onclick` for a11y

## CustomCursor

**File:** `src/lib/components/animation/CustomCursor.svelte`

The site uses a fully custom cursor ‚Äî the native OS cursor is hidden site-wide and replaced by this component.

### How it works

- `cursor: none` is set on `*,*::before,*::after` in `app.css` as an author-level direct rule ‚Äî this overrides the browser's `cursor: pointer` on `<a>` elements (which is a UA rule, always weaker than author rules)
- `CustomCursor` is rendered in both the portfolio and blog layouts; it sits at `z-index: 10000` with `pointer-events: none` so it's always above all content but never blocks clicks
- The `cursorStore` (`src/lib/stores/cursor`) holds the current variant (`'default'` | `'hover'` | `'drag'`)

### Triggering cursor variants

```svelte
<a href="/" use:cursorTarget={'hover'}>Link</a>
<div use:cursorTarget={'drag'}>Draggable</div>
```

Add `use:cursorTarget={'hover'}` to **every** interactive element (links, buttons, modal triggers). The action handles `mouseenter`/`mouseleave` and updates the store.

**For mdsvex-generated prose links** (can't use Svelte actions on markdown output):

```ts
onMount(() => {
  const links = proseEl.querySelectorAll('a');
  const enter = () => cursorStore.setVariant('hover');
  const leave = () => cursorStore.setVariant('default');
  links.forEach((l) => {
    l.addEventListener('mouseenter', enter);
    l.addEventListener('mouseleave', leave);
  });
  return () => links.forEach((l) => {
    l.removeEventListener('mouseenter', enter);
    l.removeEventListener('mouseleave', leave);
  });
});
```

### Navigation reset

SPA navigation fires before `mouseleave` on the clicked link, leaving the store in `'hover'`. Always reset in `afterNavigate`:

```ts
afterNavigate(() => {
  cursorStore.setVariant('default');
  // ... rest of afterNavigate
});
```

This is already wired in both `(portfolio)/+layout.svelte` and `blog/+layout.svelte`.

## Button (`btn`)

Not a standalone component ‚Äî a CSS class convention used across portfolio pages.

```css
.btn { /* base: padding, border-radius, font, transition */ }
.btn--primary { background: var(--accent); color: var(--bg); }
.btn--ghost { border: 1px solid var(--border); color: var(--text-muted); }
```

Hover: `translateY(-2px)` lift. Ghost hover: border + text switch to `var(--accent)`.

## Tag

Not a standalone component ‚Äî a `.tag` span class used on the work list and slug pages.

- **Work list (`/work`):** grey text/border at rest; fill-from-bottom accent color on `.work-item:hover` via `clip-path` on `::after { content: attr(data-text) }`
